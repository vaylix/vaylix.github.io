---
title: Roadmap Boundaries
description: What is implemented today, what is explicitly not, and why the docs should stay honest.
---

## Implemented Today

- single-node key/value database
- framed binary transport
- optional TLS
- mandatory auth
- encrypted snapshot + WAL durability
- audit log
- optional zstd frame compression
- atomic single-node `EXEC`

## Not Implemented Today

- replication
- sharding
- distributed transactions
- MVCC
- formal isolation levels
- multi-user ACLs

## Why This Matters

Vaylix is being shaped for those future capabilities, but the codebase should not pretend they already exist.

The documentation site should follow the same rule:

- document shipped behavior as shipped behavior
- document roadmap constraints as roadmap constraints
- avoid vague marketing language that hides technical gaps

That honesty is useful to contributors, operators, and anyone evaluating the project.
