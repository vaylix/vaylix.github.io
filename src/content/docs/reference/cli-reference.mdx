## Server Flags

### Networking and Security

| Flag | Meaning |
| --- | --- |
| `--bind` | Bind address. Default `127.0.0.1`. |
| `--port` | Port. Default `9173`. |
| `--ssl` | Enable TLS for server transport. TLS is disabled by default. |
| `--tls-cert` | PEM certificate chain, required when TLS is enabled. |
| `--tls-key` | PEM private key, required when TLS is enabled. |
| `--tls-client-ca` | PEM CA bundle used to require and verify client certificates for mTLS. |
| `--user` | Username. Default `vaylix`. |
| `--password` | Password. Default `vaylix`. |
| `--disable-auth` | Disable auth for trusted local testing only. |

### Persistence and Runtime

| Flag | Meaning |
| --- | --- |
| `--data-dir` | Persistent data directory. |
| `--wal-sync` | `buffered`, `flush`, or `sync`. |
| `--snapshot-interval-seconds` | Periodic background snapshots. |
| `--expiration-sweep-interval-seconds` | TTL sweeper interval. |
| `--idle-timeout-seconds` | Idle disconnect timeout. |
| `--audit-log-path` | Override audit log file path. |

### Guardrails and Compression

| Flag | Meaning |
| --- | --- |
| `--max-connections` | Concurrent session cap. |
| `--max-request-payload-bytes` | Max accepted payload size after framing. |
| `--max-key-bytes` | Max key size. |
| `--max-value-bytes` | Max value size. |
| `--max-keys-per-batch` | Max key count in batch commands. |
| `--max-transaction-queue-len` | Max queued commands inside `MULTI`. |
| `--requests-per-second` | Sustained per-connection rate. |
| `--request-burst` | Burst budget for rate limiting. |
| `--disable-compression` | Disable default outbound zstd frame compression. |

## Client Flags

| Flag | Meaning |
| --- | --- |
| `--url` | Full connection string, e.g. `vaylix://user:password@127.0.0.1:9173`. |
| `--host` | Host when `--url` is not used. |
| `--port` | Port when `--url` is not used. |
| `--ssl` | Enable TLS. Plain TCP is used by default. |
| `--tls-ca-cert` | Custom CA bundle for server verification. |
| `--tls-client-cert` | Client certificate chain used for mTLS. Requires TLS and `--tls-client-key`. |
| `--tls-client-key` | Client private key used for mTLS. Requires TLS and `--tls-client-cert`. |
| `--user` | Username override. |
| `--password` | Password override. |
| `--disable-auth` | Skip auth handshake for trusted local testing only. |
| `--output` | `plain`, `table`, or `json`. |
| `--disable-compression` | Disable default outbound zstd frame compression. |

## Connection String Example

```text
vaylix://vaylix:vaylix@127.0.0.1:9173?ssl=true&output=json&compression=zstd
```

Supported URL query parameters include:

- `ssl=true`
- `output=plain|table|json`
- `ca_cert=/path/to/ca.pem`
- `client_cert=/path/to/client.crt`
- `client_key=/path/to/client.key`
- `auth=false`
- `compression=none|zstd`

CLI flags take precedence over values parsed from the URL.
