## Layering

The transport crate is the shared source of truth for client and server communication.

Flow:

`Request -> transport encode -> TCP/TLS bytes -> transport decode -> Response`

## Frame Header

Each frame carries:

| Field | Type | Purpose |
| --- | --- | --- |
| `magic` | `u32` | Protocol identification |
| `version` | `u8` | Wire version |
| `flags` | `u8` | Compression and future transport flags |
| `length` | `u32` | Payload byte length |
| `checksum` | `u32` | CRC32 of the on-wire payload |

Current header length: `14` bytes.

## Request and Response Bodies

Requests:

- `request_id: UUID`
- `opcode: u8`
- request payload

Responses:

- `request_id: UUID`
- `status: u8`
- response payload

## Compression

Current outbound compression mode:

- `none`
- `zstd`

Compression is frame-level, not payload-field-level. The checksum covers the compressed bytes that were actually transmitted.

The reader checks the frame flag and transparently decompresses before request or response decoding.

## Status Codes

| Status | Meaning |
| --- | --- |
| `Ok` | Successful request |
| `Error` | Structured remote error payload |
| `NotFound` | Missing value / key |

## Remote Errors

Remote failures are structured rather than ad hoc strings:

- stable code
- friendly name
- message

That makes the wire format more suitable for future SDKs and tooling.
